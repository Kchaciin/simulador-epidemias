01 — Introducción

¿Qué es el Simulador de Epidemias?

El Simulador de Epidemias Urbanas es una aplicación web que modela la propagación de cuatro enfermedades infecciosas sobre el mapa real de San Felipe, Yaracuy. Todos sus parámetros de transmisión, mortalidad e intervención provienen de fuentes verificadas — OMS, CDC y literatura epidemiológica revisada por pares. La aplicación corre completamente en el navegador, sin servidores propios, sobre un stack tecnológico moderno y de código abierto.

2

Eficiencia de Rendimiento

ISO/IEC 25010 — §4.2  ·  Tiempo de respuesta · Utilización de recursos · Capacidad
9.0/10

"El simulador corre fluido con 300+ agentes y mosquitos — sin congelar la pantalla ni agotar la memoria."

Los agentes se dibujan en un Canvas 2D superpuesto sobre el mapa — no como elementos HTML individuales. Mover 300 divs del DOM por frame costaría ~15ms; dibujar 300 círculos en canvas cuesta ~1.5ms. Las rutas de movimiento se pre-calculan con OSRM una sola vez al iniciar y quedan en memoria, eliminando cualquier latencia de red durante la ejecución.

// Renderizado O(n) — canvas es ~10× más rápido que DOM ctx.clearRect(0, 0, w, h); for (const a of agents) { const px = map.project([a.lng, a.lat]); // geo → píxel ctx.arc(px.x, px.y, a.radius, 0, 2 * Math.PI); ctx.fill(); }

El motor soporta velocidades hasta ×20. Con este multiplicador, el ciclo completo de Tuberculosis (180 días simulados) es observable en 3.6 minutos reales — viable para una defensa de proyecto.

5

Confiabilidad

ISO/IEC 25010 — §4.5  ·  Madurez · Disponibilidad · Tolerancia a fallos
9.0/10

"El sistema produce resultados epidemiológicamente correctos de forma consistente — y cuando algo podría fallar matemáticamente, está protegido."

Un error frecuente en simulaciones es usar distribución uniforme para las transiciones de estado (muerte, recuperación). El simulador usa en cambio una distribución cuadrática con peso hacia el final de la enfermedad — que refleja el patrón clínico real donde la mayoría de eventos ocurren después de transcurrida la mayor parte del período.

// Distribución cuadrática — peso concentrado en los ticks finales f(t) = 3 · (t / T)² / T // T = d_rec × 24 ticks de enfermedad P(I → D en tick t) = α · f(t) // muerte P(I → R en tick t) = (1 - α) · f(t) // recuperación // Protección contra división por cero en R efectivo: if (infectadosResueltos === 0) return null; // → muestra "—" en UI
✅
Sin punto único de falloToda la lógica corre en el navegador. Las únicas llamadas externas (OSRM, CARTO) ocurren al inicio y sus resultados quedan en memoria. Un corte de internet durante una presentación no interrumpe la simulación.

7

Mantenibilidad

ISO/IEC 25010 — §4.7  ·  Modularidad · Reutilización · Analizabilidad · Modificabilidad
9.5/10

"Un desarrollador nuevo puede entender, modificar y extender el código sin romper el resto del sistema."

La decisión de diseño más importante es la arquitectura data-driven: el motor de simulación no conoce ninguna enfermedad en particular — solo lee parámetros de un objeto de configuración. Agregar una quinta enfermedad (Influenza, Sarampión, Ébola) requiere cero cambios en el motor.

// ❌ Anti-patrón — motor acoplado a enfermedades específicas if (disease === "covid") { p = 0.05; r = 1.5; } else if (disease === "tb") { p = 0.03; r = 2.0; } // Cada enfermedad = cambio de código // ✅ Arquitectura data-driven del simulador const p = config.p_base * K_int * K_mask * K_clima; // viene de diseases.js const r = config.r_contagion_m; // configurado, no hardcoded
MóduloResponsabilidad únicaLo que explícitamente NO hace
simulation.jsLógica de estados y ticksNo toca el DOM, no dibuja nada
renderer.jsDibujar agentes en canvasNo calcula contagios ni transiciones
diseases.jsParámetros de enfermedadesNo ejecuta ninguna lógica de simulación
charts.jsActualizar gráficas SIRNo conoce la epidemiología directamente
pathfinding.jsRutas y posición de agentesNo decide cuándo ni a dónde moverse

06 — Conclusiones

Síntesis y Puntuación Global

El Simulador de Epidemias Urbanas cumple las ocho características de calidad del estándar ISO/IEC 25010 en un nivel académico y profesional. Sus fortalezas principales son la exactitud científica de sus datos epidemiológicos (parámetros verificables de OMS y CDC), la arquitectura data-driven que permite extender el sistema sin modificar el motor, y la portabilidad absoluta propia de una aplicación web pura.
